Transformation Rules

Shift, Reflect, Scale — the three core operations that turn any known graph into a family of related curves. Master the logic, and you'll never need to memorise a transformed graph again.

Base function f(x) = x² (white) · Shifted, reflected, and scaled variants overlaid

📐 Shift (Translation)

VERTICAL SHIFT

f(x) + k
k > 0 → up, k < 0 → down

Add a constant to the output. The entire graph moves up or down without changing shape.

HORIZONTAL SHIFT

f(x - h)
h > 0 → right, h < 0 → left

Subtract from the input. The graph moves opposite to the sign: (x-2) shifts right, (x+3) shifts left.

Narration: "To shift a graph vertically, add a constant to the whole function — f(x)+2 lifts every point up by 2. To shift horizontally, replace x with (x−h). The trick is that (x−2) moves right, not left, because to get the same output as before, x must be 2 units larger. I always verify by checking the vertex or a key point."

🪞 Reflect

ACROSS X‑AXIS

−f(x)

Multiply the whole function by −1. Everything above the axis flips below, and vice versa. x‑intercepts stay fixed.

ACROSS Y‑AXIS

f(−x)

Replace x with −x. The left and right sides of the graph swap. If the function is even, there is no visible change.

Narration: "Reflection across the x‑axis negates the output: y becomes −y. Reflection across the y‑axis negates the input: x becomes −x. These are fundamentally different operations — one flips vertically, the other horizontally. For odd functions like x³, both reflections produce the same result, but for general functions like eˣ, they are completely different."

🔍 Scale (Stretch / Compress)

VERTICAL SCALE

a·f(x)
|a| > 1 → stretch, 0 < |a| < 1 → compress

Multiply the output. y‑values are scaled by a. If a is negative, it also reflects across the x‑axis.

HORIZONTAL SCALE

f(b·x)
|b| > 1 → compress, 0 < |b| < 1 → stretch

Multiply the input. This is counter‑intuitive: b=2 makes the graph twice as narrow (compressed), b=0.5 makes it twice as wide (stretched).

Narration: "Vertical scaling multiplies all y‑coordinates. Horizontal scaling multiplies x‑coordinates by 1/b — that's why f(2x) compresses the graph: to reach the same y, x needs to be half as large. The order of transformations matters: if I have −2f(3x+1)+4, I'd work from the inside out — shift, then scale, then reflect and shift vertically."

🎯 Likely Interview Questions

Q1: What is the difference between vertical and horizontal shift?+

Vertical shift adds a constant outside the function (f(x)+k), moving the graph up or down. Horizontal shift changes the input (f(x−h)), moving the graph left or right. Crucially, (x−2) shifts right, not left — this is the most common mistake.

Q2: How would you graph y = −(x−3)² + 5 starting from y = x²?+

Start with the parent parabola. Shift right by 3 (replace x with x−3) → vertex at (3,0). Reflect across x‑axis (multiply by −1) → opens downward. Then shift up by 5 → vertex moves to (3,5). The order: horizontal shift, reflection, vertical shift.

Q3: Why does f(2x) compress the graph horizontally rather than stretch it?+

Because the x‑value needed to reach a given y is halved. For f(x)=x², the point (2,4) becomes (1,4) in f(2x) because f(2·1) = f(2) = 4. The graph squeezes toward the y‑axis. It's a compression by factor 1/2.

Q4: How do you handle multiple transformations in the right order?+

Work from the inside out, following BIDMAS/PEMDAS: 1) Horizontal shift (inside the bracket), 2) Horizontal scale (multiply x), 3) Reflection & vertical scale (multiply the function), 4) Vertical shift (add constant). Always apply transformations to the x or y values in that sequence.

Q5: What transformation turns f(x) into f(|x|)?+

That's a modulus on the input. It's not a simple shift/reflect/scale; it's a piecewise transformation: for x≥0, the graph stays the same; for x<0, we take the right‑side graph and mirror it to the left, making the function even. This is a fundamentally different operation and often creates sharp corners at x=0.

Q6: If I have the graph of eˣ, how would I sketch y = 2eˣ⁻¹ − 3?+

Start with eˣ. Shift right by 1 (eˣ⁻¹). Then stretch vertically by factor 2. Then shift down by 3. The horizontal asymptote moves from y=0 to y=−3. So the graph still grows exponentially but is twice as steep and sits 3 units lower.

⚡ 30‑SECOND INTERVIEW SUMMARY

"Graph transformations are built on three operations. Shift: f(x)+k moves the graph vertically, f(x−h) moves it horizontally — remembering that (x−2) shifts right, not left. Reflect: −f(x) flips across the x‑axis, f(−x) flips across the y‑axis. Scale: a·f(x) stretches or compresses vertically; f(bx) does so horizontally, with the counter‑intuitive rule that |b|>1 compresses and 0<|b|<1 stretches. The correct order is horizontal shift, horizontal scale, reflection/vertical scale, then vertical shift. These rules allow you to sketch any transformed function from a known parent graph — a skill that signals genuine algebraic fluency far beyond rote memorisation."
